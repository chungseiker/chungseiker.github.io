네이버 블로그에서 GitHub으로

어제 진행되었던 네이버 D2 캠퍼스 세미나에서 동아리 형들과 다양한 이야기를 주고받으면서 일일커밋에 대해 이야기를 주고받았는데, 그것이 자극이 되어 블로그 서비스를 GitHub으로 이전하게 되었습니다.

사실 예전부터 워드프레스를 시도하는 등 네이버 블로그 서비스를 탈출하기 위한 시도는 여러번 있었는데, 기나긴 PHP 삽질을 겪으면서 워드프레스는 ㅌㅌ…

네이버 블로그에서 제공하는 SmartEditor 또한 지속적으로 판올림을 하면서 상당히 질 높은 수준의 포스트를 작성할 수 있게 되었지만, GitHub과 Markdown 문법을 활용한 블로깅 서비스가 상당히 수준이 높게 잘 되어있었습니다.

블로깅 서비스는 hexo를 사용하였으며, 굳이 선택하게 된 이유를 꼽자면

  1. node.js를 기반으로 하여 확장성이 용이하고, 나만의 블로그 생성에 편리하다.
  2. CS와 관련된 잡다한 내용을 정리하는데 최적화되어있다.
  3. (커리어에 도움이 될 것 같아…)

하지만 기존에 사용하던 네이버 블로그 서비스 또한 꾸준하게 적은 수의 방문자가 있으므로, 네이버 블로그에서는 정말 잡다한 내용 위주로 게시하고 GitHub 블로그에서는 개발과 관련된 포스팅을 주로 올리고자 합니다.

여튼 GitHub에서도 즐거운 삽질을… 후후
